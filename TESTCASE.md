## TC-001: Создание с валидными данными

> **Предусловие:** Сервер доступен, переменная `{{baseUrl}}` настроена
>
> **Шаги:**
> 1. Отправить POST запрос на `{{baseUrl}}/api/1/item`
> 2. В Body указаны:
> ```json
> {
>   "sellerID": 123456,
>   "name": "Test item",
>   "price": 1000,
>   "statistics": {
>     "likes": 5,
>     "viewCount": 100,
>     "contacts": 10
>   }
> }
> ```
>
> **Ожидаемый результат:**
> - Статус `200 OK`
> - Response содержит:
>   - `id` (UUID формат)
>   - `sellerId` (число)
>   - `name` (строка)
>   - `price` (число)
>   - `statistics` (объект с `likes`, `viewCount`, `contacts`)
>   - `createdAt` (дата/время)
>
> **Приоритет:** `High`

> 3. **READ by Seller**: GET `/api/1/{sellerID}/item` → проверить наличие в списке

> 6. **VERIFY**: GET `/api/1/item/{id}` → проверить `404`
>
> **Ожидаемый результат:**
> - Все шаги выполнены успешно
> - Данные консистентны на всех этапах
>
> **Приоритет:** `High`
