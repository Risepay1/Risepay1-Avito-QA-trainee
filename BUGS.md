## Баг #1: API не принимает нулевые значения в статистике объявления

> **ID тест-кейса:** TC-002  
> **Серьёзность:** `High`  
> **Приоритет:** `High`  
> **Статус:** `New`
>
> ### Краткое описание
> API возвращает ошибку `400 Bad Request` с сообщением `"поле likes обязательно"` при создании объявления с нулевыми значениями в объекте `statistics`,
> несмотря на то, что все обязательные поля присутствуют в запросе.
>
> ### Шаги воспроизведения
> 1. Отправить POST запрос на `{{baseUrl}}/api/1/item`
> 2. В Body (raw JSON) указать:
> ```json
> {
>   "sellerID": 234567,
>   "name": "Test with zeros",
>   "price": 1000,
>   "statistics": {
>     "likes": 0,
>     "viewCount": 0,
>     "contacts": 0
>   }
> }
> ```
> 3. Headers: `Content-Type: application/json`, `Accept: application/json`
> 4. Нажать **Send**
>
> > ### Ожидаемый результат
> - **Status:** `200 OK`
> - Объявление успешно создано, нулевые значения корректно сохраняются
>
> ### Фактический результат
> - **Status:** `400 Bad Request`
> - **Response:**
> ```json
> {
>   "result": {
>     "message": "поле likes обязательно",
>     "messages": {}
>   },
>   "status": "400"
> }
> ```
>
> ### Окружение
> - **Host:** `https://qa-internship.avito.com`
> - **Endpoint:** `POST /api/1/item`

## Баг #2: API принимает отрицательную цену товара

> **ID тест-кейса:** TC-013  
> **Серьёзность:** `Medium`  
> **Приоритет:** `High`  
> **Статус:** `New`
>
> ### Краткое описание
> API успешно создаёт объявления с отрицательной ценой (`price: -100`), что противоречит бизнес-логике — цена товара не может быть отрицательной.
>
> ### Шаги воспроизведения
> 1. Отправить POST запрос на `{{baseUrl}}/api/1/item`
> 2. В Body (raw JSON) указать:
> ```json
> {
>   "sellerID": 345678,
>   "name": "Negative price",
>   "price": -100,
>   "statistics": {
>     "likes": 1,
>     "viewCount": 1,
>     "contacts": 1
>   }
> }
> ```
> 3. Headers: `Content-Type: application/json`, `Accept: application/json`
> 4. Нажать **Send**
>
> > ### Ожидаемый результат
> - **Status:** `400 Bad Request`
> - Валидация отклоняет запрос с отрицательной ценой
>
> ### Фактический результат
> - **Status:** `200 OK`
> - **Response:**
> ```json
> {
>   "status": "Сохранили объявление - 3827cac9-144b-48b4-b446-848030e377d2"
> }
> ```
> - Объявление успешно создано с ценой `-100`
>
> ### Окружение
> - **Host:** `https://qa-internship.avito.com`
> - **Endpoint:** `POST /api/1/item`

## Баг #3: API принимает отрицательный sellerID

> **ID тест-кейса:** TC-204  
> **Серьёзность:** `Low`  
> **Приоритет:** `Medium`  
> **Статус:** `New`
>
> ### Краткое описание
> API успешно возвращает данные при запросе с отрицательным `sellerID` (-123), что не соответствует требованиям (рекомендуемый диапазон 111111–999999).
>
> ### Шаги воспроизведения
> 1. Отправить GET запрос на `{{baseUrl}}/api/1/-123/item`
> 2. Headers: `Accept: application/json`
> 3. Нажать **Send**
>
> > ### Ожидаемый результат
> - **Status:** `400 Bad Request` — валидация отклоняет некорректный `sellerID`
> - **ИЛИ** `200 OK` с пустым массивом `[]` (если продавец не найден)
> - 
> ### Фактический результат
> - **Status:** `200 OK`
> - В ответе присутствуют объявления с `"sellerId": -123` и `"price": -100`
>
> ### Окружение
> - **Host:** `https://qa-internship.avito.com`
> - **Endpoint:** `GET /api/1/:sellerID/item`
